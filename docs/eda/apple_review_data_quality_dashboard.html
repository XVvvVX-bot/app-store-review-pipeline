Known Limitations

This dashboard separates what the pipeline has collected from what the public source can prove.

  • The source is Apple-hosted public web catalog review data exposed to the App Store web experience. It is not the App Store Connect Customer Reviews API, does not use app-owner credentials, and does not come with a third-party bulk-ingestion SLA.
  • Historical completeness is empirical per app-country-source scope. A scope is only treated as exhausted when pagination reaches no_next_href; page cap, time budget, overlap, final non-200, or fetch-error stops mean the stored review count is still a lower bound.
  • Incremental runs rely on stable review keys and Postgres upserts. New rows and updated rows can be captured across repeated runs, but source ordering, removed reviews, and Apple response-shape changes should be monitored through page, retry, and terminal-reason metrics.
  • The public payload does not currently include every owner-API field. Version, vote sum, vote count, and similar App Store Connect-style fields are intentionally excluded from the production review schema until they appear in the public response.
